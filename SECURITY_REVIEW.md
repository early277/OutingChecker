# セキュリティ / プライバシー / 構造レビュー

確認日: 2026-05-01
対象: `OutingChecker` リポジトリ（iOS/Watch app + Widget）

## 結論（サマリ）

- **重大な脆弱性（外部送信、認証情報漏えい、任意コード実行）**は確認されませんでした。
- 一方で、運用上のリスクとして以下を確認しました。
  1. **WCSession.delegate の競合**による同期不整合リスク（構造上の問題）
  2. **ローカル保存データの平文保存**（プライバシー強化余地）
  3. **`exit(0)` による強制終了**（可用性/品質の問題）

---

## 1) セキュリティ観点

### 確認したこと

- コード中に API キー、トークン、シークレット直書きは見当たりません。
- 外部サーバー通信（HTTP クライアント、解析 SDK、広告 SDK）は見当たりません。
- 共有権限は App Group のみで、過剰な権限付与は見当たりません。

### 懸念点

- **Watch から受信したデータをそのまま保存**しており、サイズ上限や項目検証がありません。
  - 通常は同一開発者の companion 間通信なので脅威は限定的ですが、データ肥大化時のパフォーマンス低下や意図しない上書きは起こりえます。

### 推奨対応

- `ChecklistItem` 配列の受信時に、件数上限・タイトル長上限・重複 ID の正規化を追加。
- デコード失敗や異常件数時のログ（開発ビルド限定）を追加。

---

## 2) プライバシー観点

### 確認したこと

- データは `UserDefaults`（App Group）保存で、外部送信なし。
- 保存項目はチェックリスト用途に必要な最小限（タイトル、状態、並び順、リセット規則）。

### 懸念点

- **保存データは平文**のため、端末バックアップ・フォレンジック文脈での閲覧耐性は高くありません。
  - 現状のデータ種別は高機微とは言いにくいものの、項目名次第で個人性が上がる可能性があります。

### 推奨対応

- プライバシーポリシーに「端末内平文保存」の明記を検討。
- 将来的に機微データ想定がある場合、保存先を Keychain / FileProtection 付きファイルへ段階移行。

---

## 3) 構造・安定性（メモリリーク含む）

### 確認したこと

- 顕在的なメモリリーク（循環参照、未解放タイマー/通知監視）を起こす実装は目立ちません。
- ただし、以下の**設計上の不整合**があります。

### 主要懸念（優先度: 中）

1. **WCSession.delegate の二重管理**
   - `WatchSessionBridge` と `IOSWatchSyncManager` がどちらも `WCSession.default.delegate` を設定。
   - `WCSession` の delegate は単一のため、後勝ちで上書きされ、片方の受信/送信フローが不安定化する可能性。

2. **アプリ強制終了 (`exit(0)`)**
   - iOS アプリのユーザー操作でプロセス終了はプラットフォーム慣習に反し、審査/UX/データ整合の観点で非推奨。

### 推奨対応

- WCSession を単一マネージャに集約し、Store からは抽象インターフェース経由で同期要求する構成へ。
- `exit(0)` は削除し、代替として UI ガイダンス（「ホームに戻る」）を表示。

---

## 4) 追加で実施した静的チェック

- `rg` による以下の探索を実施:
  - 通信/URL/危険 API の利用有無
  - 永続化、Watch 同期、通知購読、終了処理
- 重大なハードコード秘密情報や外部送信処理は未検出。

---

## 優先度付きアクションプラン

- **P1（早め）**: WCSession delegate の単一化。
- **P2（早め）**: `exit(0)` 廃止。
- **P3（任意）**: 受信データ検証（件数・文字数・重複）。
- **P3（任意）**: プライバシーポリシーへ保存形式の明確化。
